反复编辑失败时注入提示词，要求ai使用该工具

该工具要求llm以输出的方式给出完整的修改意见，包括修改的行数范围(允许放大，后续处理时可以根据文件长度调整)和修改指导，但禁止包含具体的修改内容，以免llm的幻觉影响后续编辑。

该工具封装一次llm请求(使用qwne3-32b-nothinkging级别的模型)，要求llm根据指导意见完成修改，仅提供diff工具，以保持干净的上下文。
如果修改意见不明确，则直接使用单一的json，内部列出修改意见的不明确之处，并为主llm注入提示词，要求llm重新使用，作为工具返回结果。如果成功则系统给出diff作为工具返回结果(可以考虑加上lsp诊断信息)

设计思路：连续2次编辑失败后让llm选择该工具，避免主llm死循环或调用重写工具执行破坏性修改
工具内不包装agent：单轮对话能提供最干净的上下文，避免子llm也死循环
